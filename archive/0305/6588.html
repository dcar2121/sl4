I'll go on record here and say that does Forth has certain inherent
advantages; from a topological point of view, it basically is the
functionality layer that you would have to build yourself in a higher-level
language to enable code that can change itself on the fly. I have used Forth
myself in situations where it (or something similar) was necessary due to
the requirement to create and integrate arbitrary new sections of code with
arbitrary APIs without stopping the overall software package from running.
